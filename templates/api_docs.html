RS Scanner 2 API Documentation

RESTful API for accessing stock market relative strength data

Base URL: Loading...
Version: v1
Authentication: None (Public API)
Rate Limiting: None currently

Overview

The RS Scanner 2 API provides programmatic access to relative strength (RS) data for US stocks. All responses are in JSON format with a consistent structure.

Response Format

{
  "status": "success",
  "data": { ... },
  "meta": {
    "timestamp": "2026-01-29 15:16:00 UTC",
    "total_count": 1159
  },
  "error": null
}

Endpoints

GET /api/v1/all

Get all data including Individual RS, WRS, and Market Condition.

Example Request

fetch('/api/v1/all')
  .then(res => res.json())
  .then(data => console.log(data));

Response Fields

  • data.data - Array of individual stock data
  • data.wrs_data - Array of WRS sector/industry data
  • data.market_condition - Current market condition text
  • data.last_updated - Data update timestamp

GET /api/v1/individual

Get individual stock RS data with optional filters.

Query Parameters

Parameter Type Description Example
ticker string Filter by ticker symbol ?ticker=AAPL
sector string Filter by sector ?sector=Technology
rs_min float Minimum RS(6mo) value ?rs_min=0.1
limit integer Limit number of results ?limit=10

Example Request

curl "https://your-url.vercel.app/api/v1/individual?sector=Technology&limit=5"

Python Example

import requests

response = requests.get('https://your-url.vercel.app/api/v1/individual', 
                       params={'sector': 'Technology', 'limit': 5})
data = response.json()
print(data['data'])

GET /api/v1/wrs

Get Weighted Relative Strength data by sector/industry.

Query Parameters

Parameter Type Description Example
sector string Filter by sector ?sector=Healthcare
min_count integer Minimum stock count ?min_count=5

Response Fields

  • Sector - Sector name
  • Industry - Industry name
  • Count - Number of stocks in group
  • WRS_6mo_MD - Median RS(6mo) value
  • WRS_MD_Rank_Pct - Percentile rank of median

GET /api/v1/todays-list

Get filtered stocks based on predefined criteria (strong sectors + top RS stocks).

Filter Criteria:
Sectors: Count ≥ 2, WRS rank ≤ 30%, WRS_MD rank ≤ 40%
Stocks: RS rank ≤ 20% within qualified sectors

Example Request

fetch('/api/v1/todays-list')
  .then(res => res.json())
  .then(data => {
    console.log(`Found ${data.meta.total_count} stocks`);
    console.log(`In ${data.meta.qualified_sectors_count} sectors`);
  });

GET /api/v1/market-condition

Get current market condition status.

Example Response

{
  "status": "success",
  "data": {
    "market_condition": "good 인규바보",
    "last_updated": "2026-01-29 15:16:00 UTC"
  },
  "meta": {
    "timestamp": "2026-01-29 15:16:00 UTC"
  },
  "error": null
}

GET /api/v1/ticker/<symbol>

Get detailed information for a specific ticker.

URL Parameters

  • symbol - Ticker symbol (e.g., AAPL, NVDA)

Example Request

GET /api/v1/ticker/NVDA

Example Response

{
  "status": "success",
  "data": {
    "Ticker": "NVDA",
    "Price": 145.23,
    "Market Cap": "3.56T",
    "RS_6mo": 0.4523,
    "RS_Rank_Pct": 2.15,
    "RS_3mo": 0.3214,
    "RS_1mo": 0.1234,
    "50DIV": 12.34,
    "Sector": "Technology",
    "Industry": "Semiconductors"
  },
  "error": null
}

Error Response (Not Found)

{
  "status": "error",
  "data": null,
  "error": {
    "code": "NOT_FOUND",
    "message": "Ticker 'XYZ' not found"
  }
}

Error Handling

All errors follow a consistent format:

{
  "status": "error",
  "data": null,
  "meta": { ... },
  "error": {
    "code": "ERROR_CODE",
    "message": "Human readable error message"
  }
}

CORS

CORS is enabled for all API endpoints. You can make requests from any origin.

Rate Limiting

Currently, there is no rate limiting. Please use the API responsibly. Rate limiting may be implemented in the future.

Data Updates

Data is updated daily after US market close (approximately 21:00 UTC / 6:00 AM KST).
